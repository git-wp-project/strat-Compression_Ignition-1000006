为何此框架与众不同

超越传统均线交叉策略

大多数零售交易方法依赖滞后指标或简单的趋势跟随规则，在区间震荡或低波动率环境中表现不佳。压缩点火协议采用另类思路，专门针对波动率收缩阶段（压缩期），随后捕捉高概率的动能点火突破。

该体系灵感来源于2026年在TradingView上流行的机构级概念，包括波动率门控策略和流动性驱动突破，但以清晰、规则化、完全可执行的方式实现，适合在MT4/MT5平台部署。

核心差异化优势
波动率优先逻辑
  • 等待真正的压缩，而非始终持仓
  • 多重过滤器 confluence（波动率 + 动能 + 趋势强度）
  • 基于真实账户风险的动态仓位管理
  • 清晰、不重绘的已完成K线信号

核心交易逻辑

精确、规则化的条件，用于识别高概率参与机会。

做多 / 看涨参与
1. 波动率压缩

布林带宽度达到历史低位（当前宽度 < 20根K线平均宽度的80%）。

2. 突破触发

价格在信号K线收盘于布林带上轨之上。

3. 动能确认

RSI(14) > 50 且 ADX(14) > 25 并呈上升趋势。

4. 趋势方向过滤

+DI > -DI，确认看涨趋势强度。

做空 / 看跌参与
1. 波动率压缩

与做多相同的压缩条件。

2. 突破触发

价格收盘于布林带下轨之下。

3. 动能确认

RSI(14) < 50 且 ADX(14) > 25 并呈上升趋势。

4. 趋势方向过滤

-DI > +DI，确认看跌趋势强度。

风险管理
止损位置
入场价 ± 1.5 × ATR(14)

根据波动率动态调整止损距离，避免在噪音市场中过早出局。

收益目标
止盈目标
入场价 ± 3.0 × ATR(14)

目标风险回报比约为 1:2，捕捉点火后的延伸行情。

仓位管理
动态风险控制
1% 每笔参与

手数自动计算，使止损时的最大亏损等于当前账户余额的1%。专业的资金保护规则。

完全可定制

参数名称 默认值 说明 影响级别

RSI周期 / 过滤阈值 14 / 50 动能过滤器，确认方向偏好 过滤器
